## `data/dashboards/<slug>.yaml`

One file per board; the filename is the board's slug, label, and URL
segment. `routines` is reserved (it's the pool view's URL segment).

```yaml title="data/dashboards/corza.yaml"
section: Clients # optional — rail section this board files under
grid:
  columns: 4 # 1–6, default 4
  rowHeight: 150 # px per row unit, default 150
  width: fixed # fixed | wide, default fixed
widgets:
  - routine: corza-pulse
    position: { col: 1, row: 1 }
    size: { cols: 2, rows: 1 }
```

### Fields

| Field                | Type             | Rules                                                                                                                                                                                           |
| -------------------- | ---------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `section`            | string           | Optional, ≤ 40 chars. Boards sharing a value cluster under one rail sub-heading; absent → the board leads the repo's unlabeled section. Order across sections comes from `repo.yaml`, not here. |
| `grid.columns`       | int              | 1–6, default 4. The board's desktop resolution; tablets render 2, phones 1.                                                                                                                     |
| `grid.rowHeight`     | int              | Default 150. Row unit in px — board density.                                                                                                                                                    |
| `grid.width`         | enum             | `fixed` centers the board at reading width; `wide` fills a large monitor.                                                                                                                       |
| `widgets[].routine`  | slug             | The routine whose artifact this cell renders.                                                                                                                                                   |
| `widgets[].position` | `{ col, row }`   | 1-indexed. Rows are unbounded.                                                                                                                                                                  |
| `widgets[].size`     | `{ cols, rows }` | `cols` 1–6, `rows` 1–12.                                                                                                                                                                        |

One cross-field invariant: a widget must fit the board it lives on —
`position.col + size.cols − 1 ≤ grid.columns`. A hand-authored layout that
overflows the grid is rejected at parse, never silently clipped at render.

### Size presets

The add-widget wizard offers named shapes instead of raw steppers —
intents, not dimensions (columns clamp to the board's own count at pick
time):

| Preset   | Size  |
| -------- | ----- |
| `small`  | 1 × 1 |
| `medium` | 2 × 2 |
| `wide`   | 4 × 1 |
| `tall`   | 2 × 3 |
| `hero`   | 4 × 3 |

## `data/repo.yaml`

Optional repo-level display metadata. A repo without one renders under its
short repo name.

```yaml title="data/repo.yaml"
name: Form Factory
sections: [Team, Clients, Projects]
```

| Field      | Type   | Rules                                                                                                                                                                                                                |
| ---------- | ------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `name`     | string | Optional, ≤ 50 chars, non-blank. Relabels the repo's rail group for everyone who reads the repo. UI only — URLs, prompts, and payloads never use it; rail order keeps sorting by slug.                               |
| `sections` | list   | Optional. Order of the rail's board sections. Membership rides on each board's `section:`; a section a board names but this list omits sorts after, alphabetically; a listed name no board uses contributes nothing. |

A malformed or missing file degrades to defaults — "no display name", "no
order" — never a failed rail.

## Legacy keys

Files written before the section rename may carry `group:` / `groups:`.
The parse boundary renames them to `section:` / `sections:` when the new
key is absent; serialization only ever emits the new keys, so the first
in-app edit rewrites a file forward. No migration required.
